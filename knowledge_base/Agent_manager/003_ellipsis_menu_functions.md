# Agent Managerの右上メニュー（3点リーダー）の機能解説

Agent Managerウィンドウの右上にある「...」（3点リーダー）アイコンから開くメニューの各機能について解説します。

## 1. 主な各ツールの機能と目的

- **Review Changes（変更のレビュー）:**
  AIエージェントが提案した、または実行したソースコードの「変更点（差分）」を確認するための機能です。AIがどこを書き換えたのかを視覚的にチェックし、安全にコードを反映・管理する際に使用します。

- **Open Editor (⌘E)（エディターを開く）:**
  現在AIと一緒に作業しているファイルや関連するコードを、メインのコードエディター領域で開く・フォーカスするためのショートカットです。AIとのチャット画面から、直接コードの編集画面へすばやく移りたい時に使います。

- **Toggle Terminal (⌘J)（ターミナルの表示/非表示）:**
  画面下部などに表示される「ターミナル（コマンドライン）」パネルの表示・非表示を切り替えます。AIが自動でコマンドを実行している様子を確認したり、自分自身で手動コマンドを打ち込んだりする際に素早くアクセスできます。

- **Follow Along with Agent（エージェントの作業に追従する）:**
  これをオンにすると、AIエージェントが背後で「どのファイルを読んでいるか」「どの部分のコードを探しているか」に合わせて、あなたの画面（エディター）も自動的に切り替わり、スクロールして追いかけてくれます。AIが今何を考えてどこを見ているのかを、文字通り「横に座って一緒に見る（ペアプログラミングする）」体験ができる非常に強力な機能です。

## 2. 具体的な利用シーン（Use Cases）

- **シーンA（コードの安全性確認）:** AIに大きなリファクタリングを任せた後、`Review Changes` を開いて、意図しない場所が消されていないか確認してから保存する。
- **シーンB（AIの思考プロセスの学習）:** AIに難しいバグの調査を依頼した際、`Follow Along with Agent` をオンにしておくことで、AIが「どのエラーログを見て、どのファイルに原因を探しに行っているのか」というプロのデバッグ手順をリアルタイムで観察し、学習する。

## 3. 注意点・ベストプラクティス

- `Follow Along with Agent` を有効にしている最中は、画面が自動で切り替わるため、あなた自身が別ファイルで手動コーディングをしていると邪魔に感じることがあります。AIに完全に調査・作業を「お任せ」している時のみオンにするのがおすすめです。
