# Agent Managerの便利な設定とカスタマイズ（修正版）

Agent Manager（AIエディタ環境）の使い勝手をさらに向上させるための代表的な設定変更について解説します。特に、AIによる自律的な作業をよりスムーズに行う自動許可設定に焦点を当てます。

## 1. チャット送信のショートカットについて（重要・現在の仕様）

多くの場合、長文を入力する際に「Enter」での誤送信を防ぎ、「Cmd+Enter」で送信したいという要望がありますが、**現在のAgent Managerの仕様・インターフェースでは、送信キーボードショートカットを独自にカスタマイズ（Cmd+Enterに変更）する機能は提供されていません。**

キーボードショートカット設定（`Cmd+Shift+P` -> `Open Keyboard Shortcuts`）等から検索しても、Agent Managerのメインチャット入力をフックする設定項目は露出していません。

### 対策・ベストプラクティス

現状では、Agent Manager内で長文のプロンプト（指示）を作成する際は、誤送信を防ぐために以下の工夫を行うのが安全な運用となります。

- **外部エディタでの下書き:** メインのコードエディタ領域でマークダウンファイルやテキストファイル（例：`prompt_draft.md`）のタブを開き、そこでプロンプトを作成・推敲してから、Agent Managerのチャット欄にコピー＆ペーストして送信する。
- 「Shift + Enter」が改行として割り当てられている場合は、意識的にそちらを使う。

## 2. AIのエラー修正・コマンド実行時の「確認プロンプト（許可）」の省略

自動で作業を進めてほしいのに、ファイル作成やコマンド実行のたびに「許可（Allow）」を押さなければならないのは非常に不便です。これを省略し、AIをより自律的に動かすための設定があります。

### 設定の仕組み（Yoloモード / Auto-Approve）

AIエディタには、AIが自律して行動する際の承認ステップをスキップするモードが備わっています。（「Yoloモード」と呼ばれることもあります）

### 変更手順

1. エディタの設定画面（右下の歯車アイコン等）を開きます。
2. 検索バーで `Agent` や `Auto Approve`、`Yolo` といったキーワードで検索します。
3. 以下のいずれかの設定が見つかった場合、オンに変更します環境により名称が異なります）。
   - **Auto Run Commands** （コマンドの自動実行を許可する）
   - **Auto Apply Changes** （ファイル変更や作成を自動で行う）
   - **Yolo Mode**

### 注意点・ベストプラクティス

- **メリット:** これをオンにすることで、AIはエラーを見つけた際に自分でファイルを書き換え、ビルドを再実行し、またエラーが出れば直す…というループを「あなたが許可ボタンを押さなくても」全自動で完結できるようになります。
- **デメリット（リスク）:** 意図しないファイルの完全書き換えや、危険なコマンドの実行が行われる可能性がゼロではありません。
- **推奨の運用:** 開発の初期段階など「AIに丸投げして作ってもらう」時はオンにして生産性を高め、リリース前の「絶対に変なコードを混ぜてほしくない」時はオフにして1つずつ確認する、という使い分けが最強のベストプラクティスです。
