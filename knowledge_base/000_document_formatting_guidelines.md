# ドキュメント作成フォーマットガイドライン

このドキュメントは、AIが新しくKnowledgeベースやドキュメントを作成・まとめる際に従うべき共通のフォーマットとルールを定義するものです。

## 1. ドキュメントの基本構造

新しいナレッジドキュメントを作成する際は、必ず以下の構造に従ってください。

### 1-1. タイトルと概要

- ドキュメントの先頭には「#（見出し1）」で簡潔で分かりやすいタイトルをつけます。
- タイトルの直後に、このドキュメントが何について書かれているのか（目的・対象）を1〜3文で簡潔に記載します。

### 1-2. 見出しの階層

- 大項目は「##（見出し2）」から始めます。
- 中項目は「###（見出し3）」、小項目は「####（見出し4）」と順に階層を下げて使用します。
- 階層構造は論理的に破綻しないように整理してください。

### 1-3. 本文の書き方

- **箇条書きの活用:** 3つ以上の並列する情報や手順を説明する場合は、文章ではなく箇条書き（`-` または `*`）を使用し、視認性を高めます。
- **強調表現:** 重要なキーワード、注意点、結論となる部分は **太字（`**`）\*\* を使って強調します。
- **コード・ファイル名:** ファイル名、フォルダ名、短いコードスニペット、特定のコマンドなどは、バッククォート（`` ` ``）で囲んでコード記法にします。

## 2. 内容に関するルール

### 2-1. 簡潔さと具体性

- 冗長な表現（「〜という風に考えられます」など）は避け、簡潔かつ断定的なトーン（「〜です」「〜します」）で記述します。
- 抽象的な概念を説明する場合は、必ず「具体的な利用シーン」や「例」を併記してください。

### 2-2. 構造化

- 長い説明になる場合は、表組み（Markdownテーブル）や、見出しによる段落分けを積極的に行い、情報を構造化します。

## 3. テンプレート例

以下は、新しい機能を解説する際の基本的なテンプレートです。

```markdown
# [機能・ツールの名前]の概要と使い方

[この機能が何を解決するか、どのような目的で使うかの簡潔な概要（2〜3行）]

## 1. 主な特徴・機能

- **[特徴1の名前]:** [簡潔な説明]
- **[特徴2の名前]:** [簡潔な説明]
- **[特徴3の名前]:** [簡潔な説明]

## 2. 具体的な利用シーン（Use Cases）

- **シーンA:** [どのような時に使うか、具体例]
- **シーンB:** [どのような時に使うか、具体例]

## 3. 注意点・ベストプラクティス

- [使用する上での注意点など]
- [推奨される使い方など]
```

## 4. このファイルの使い方（AIへの指示）

ユーザーから「新しく〇〇についてまとめて」と指示された場合、AIはこのドキュメント（`000_document_formatting_guidelines.md`）のフォーマットとルールを参照し、指定された内容を構造化して出力してください。
