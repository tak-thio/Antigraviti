# Antigravitiのコア機能：ワークフロー・Skills・MCPの違いと役割

AIエージェント（Antigraviti）の能力を引き出し、作業を効率化するための3つの重要な機能についてのまとめ。

## 1. ワークフロー (Workflows)：「作業手順書」

- **概要:** デプロイやテストなど、決まった一連の作業ステップを定義・自動化するための機能。
- **役割:** 「〇〇をするには、まずAを実行し、次にBを確認し、最後にCをする」といった連続したステップ（手順）に特化している。AIはこれを見ることで、迷わず正確に作業を進めることができる。
- **設定:** プロジェクトごとに設定が必要（推奨）。プロジェクト内の `.agents/workflows/` ディレクトリにMarkdown形式（`.md`）で配置する。

## 2. Skills (スキル)：「AIの専門知識」

- **概要:** AIエージェントに特定の複雑なタスクのやり方や、独自のノウハウを教え込むためのスキルセット。
- **役割:** 指示書（プロンプト）、サンプルコード、専用スクリプトなどをひとまとめにし、AIの能力（特定のフレームワークの書き方や独自ルールの遵守など）を拡張する。
- **設定:** プロジェクト固有、または個人環境全体で汎用的に設定可能。

## 3. MCP (Model Context Protocol)：「外部接続の橋渡し」

- **概要:** AIと外部の世界（ツールやデータ）を安全に繋ぐための共通の通信規格。
- **役割:** AIがPCのローカルファイルシステム、データベース、GitHub、Slackなどの外部ツールやリソースに直接アクセスして操作できるようにする。AIの「手足」となる機能。
- **設定:** 環境や連携したいツールごとに設定する。
