# MCP（Model Context Protocol）の総まとめ

AIエージェント（Antigraviti等）と外部の世界を繋ぐための画期的な仕組みであるMCPについての総まとめです。

## 1. MCPの基本概念と役割

- **正体**: AIに「手足」を与え、PCのローカル環境や外部サービス（データベース、Notion、Slack、GitHubなど）と直接やり取りさせるための**安全で統一された通信規格（プロトコル）**。
- **目的**: AIをチャット画面という「隔離された箱」から解放し、自律的に外部ツールを操作してタスクを完遂できるようにすること。

## 2. 一般的なAPIとの決定的な違い

APIが「人間（プログラマー）」のための仕様であるのに対し、MCPは最初から**「AI（LLM）自身」が使うために作られた**という点で、以下のような革命的な違いがあります。

| 特徴             | 一般的なAPI                                    | MCP (Model Context Protocol)                                                                                                                 |
| :--------------- | :--------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------- |
| **対象者**       | 人間・プログラム                               | **AI（LLM）自身**                                                                                                                            |
| **使い方の理解** | AIは事前にドキュメントを読み学習する必要がある | **Discovery機能**: 繋がった瞬間にサーバーがAIへ「私は〇〇です。こういう機能（ツール）が使えます」と**自動で教える（Skillの自動付与に近い）** |
| **フォーマット** | サービスごとにバラバラ                         | 世界中のツールが**1つの共通言語・規格で統一**                                                                                                |
| **セキュリティ** | 直接キーを渡すため広範な危険が伴う             | MCPサーバーが間に入り、**「読み取り専用」などの厳密な権限制御（隔離）**が可能                                                                |

## 3. ユーザーの素晴らしい比喩（本質的理解）

MCPの最大の特徴である「Discovery機能（自動自己紹介）」は、以下のように表現することができます。

> **「外部と繋がった瞬間に、先ほどのSkills（使い方やルール）を相手から渡してもらえるようなイメージ」**

API通信をゼロから構築する手間がなくなり、MCPのケーブルを挿したその1ミリ秒後に、AIは相手のツールに必要な「完全なスキルと使用許可」をインストールされた状態になります。これがMCPの持つ真髄です。

## 4. 三大要素（ワークフロー・Skills・MCP）の組み合わせ

これら3つを組み合わせることで、AIエージェントの真価が発揮されます。

1.  **ワークフロー**: 「いつ」「どういう順序で作業するか」（**プロセスの自動化**）
2.  **Skills**: 「どんなルールや専門知識で実行するか」（**AIの脳・ノウハウ**）
3.  **MCP**: 「どこからデータを読み、どこへ出力するか」（**AIの手足・外部連携**）

**【究極の自動化パイプラインの例】**
「**ワークフロー**で1日の終わりに起動し、**MCP**を使ってPCの作業履歴とGitHubのコミット履歴を自動で読み込み、**Skills**に定義された『自社の完璧な日報フォーマットルール』に則ってレポートを作成し、再び**MCP**を使って自動でNotionの指定ページに書き込む」
