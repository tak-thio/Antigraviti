# コードのスナップショット作成

このワークフローは、大きな変更やリファクタリングを始める前に、現在の安全なコード状態を一時保存（コミット）するためのものです。問題が発生した場合にすぐに元の状態に戻せる「安全地帯」を作ります。

## 1. 状態の確認とコミットの生成

まず、現在のGitステータスと未コミットの変更内容を確認します。

## 2. 一時保存の実行

AIが直近の変更内容から適切な短いコミットメッセージを生成し、以下のコマンドを実行してコミットします。

// turbo-all

```bash
git add .
git commit -m "chore: snapshot - [AIが生成した現在の状態の短い要約]"
```

## 3. 完了の報告

スナップショットの作成が完了したら、ユーザーに「安全なセーブポイントが作成されました。リファクタリングなどの危険な変更を始めても大丈夫です！」と報告します。メッセージには生成したコミットのハッシュを含めてください。
