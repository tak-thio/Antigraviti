# 作業進捗の自動記録とGitHubへのプッシュ（バイブコーディング用）

このワークフローは、AI（Antigraviti）との対話を通じた開発（バイブコーディング）において、ユーザーが「区切り」や「締め」と指示するだけで、AIがこれまでの会話履歴から作業内容を自動でまとめ、進捗記録からGitHubへのプッシュまでを全自動で完了させる手順です。

## 1. コンテキストの解析と情報の自動抽出（AIのタスク）

ユーザーから「区切り」「締め」「今日の作業終わり」などの指示を受けたら、AIはこれまでの直近の会話履歴や作業ログを振り返り、以下の情報を**自律的に生成・抽出**してください。

- **作業進捗**: 今回のセッションで達成した内容、作成や変更を行ったファイルなど。
- **残りのタスク**: 会話の流れから推測される、次回以降に行うべき作業。
- **その他・メモ**: 作業中の気づき、発生したエラー、決定事項など。

## 2. 進捗記録ファイルの更新

`knowledge_base/progress_log.md` に対して、抽出した内容を以下のフォーマットで追記します。ファイルが存在しない場合は新規作成します。

フォーマット:

```markdown
## YYYY-MM-DD HH:MM

### 作業進捗

- [AIがまとめた作業内容1]
- [AIがまとめた作業内容2]

### 残りのタスク

- [AIが推測した残りのタスク]

### その他・メモ

- [AIがまとめたメモ]
```

## 3. (条件付き) 日報の作成

ユーザーの指示が「締め」や「今日の作業は終了」など、1日を終えるニュアンスの場合のみ、`knowledge_base/daily_reports/YYYY-MM-DD.md` を作成します。（ディレクトリがない場合は作成）

日報フォーマット:

```markdown
# 日報: YYYY-MM-DD

## 本日の成果

- [本日の作業全体のまとめ]

## 明日の予定

- [AIが推測した明日のタスク]

## 課題・メモ

- [特筆すべき事項]
```

## 4. GitHubへのプッシュ

記録が完了したら、作業内容を端的に表すコミットメッセージをAIが自動生成し、プッシュを実行します。

// turbo-all

```bash
git add .
git commit -m "docs: [AIが生成した作業内容の短い要約] (YYYY-MM-DD HH:MM)"
git push
```

※ コミットメッセージの例: `git commit -m "docs: ワークフローの自動記録機能の追加 (2026-02-20 18:00)"`

## 5. 完了の報告

プッシュ完了後、ユーザーに以下の内容を報告してください。

- **AIが自動で記録した内容のサマリー**（どのような内容で記録したか、ユーザーが確認できるようにする）
- 更新したファイル（`progress_log.md` や日報など）
- プッシュが成功した旨
