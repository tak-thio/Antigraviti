# 機能のひな形作成 (Scaffold)

このワークフローは、新しい機能や画面を作り始める際に、必要なファイル群（コード、スタイル、テストなど）を一気に自動生成し、記述の手間を省くためのものです。

## 1. ユーザーからの要件確認

ユーザーから `/scaffold [機能名やコンポーネント名]` という指示を受け取った後、AIは作りたいものの種類（Web系ならReact/Vue/HTML+CSSなど、別の言語ならそれにあった構成）を現在のプロジェクト構成から推測します。

## 2. ファイル構成の生成

指示された機能に必要なファイル群を計画します。
例（Webコンポーネントの場合）:

- UIコンポーネントファイル (`.js`, `.jsx`, `.tsx`, `.html` など)
- スタイルファイル (`.css`, `.scss` など)
- テストファイル (`.test.js` など)

## 3. ファイルの実体化

AIは適切なディレクトリにファイルを作成し、必要最低限の初期コード（インポート文、関数定義、エクスポートなど）を書き込みます。

## 4. 完了報告

作成したファイルのパス一覧と、「ひな形の作成が完了しました。それでは本格的な実装に進みましょう」というメッセージをユーザーに伝えます。
