# AIによる自律的なコードレビュー

このワークフローは、ひと段落したコードに対してAIが自己点検を行い、改善点やセキュリティ上の懸念がないかを確認する手順です。

## 1. 変更点の抽出

AIは直近で追加・変更されたファイルの内容や、`git diff` の結果を読み込みます。

## 2. コードレビューの実施

以下の観点でAIがコードをチェックします。

- **セキュリティ**: 脆弱性になりうる箇所はないか
- **パフォーマンス**: 無駄な処理や最適化できる箇所はないか
- **可読性・保守性**: 変数名や関数名は適切か、より簡潔に書ける部分はないか
- **エラー処理**: 例外時のハンドリングは十分か

## 3. レビュー結果の提示

チェックした結果を以下のフォーマットでまとめてユーザーに提示します。
特に問題がない場合は、「現状大きな問題は見当たりません」と報告します。

```markdown
## コードレビュー結果

### 🟢 良い点

- [AIが評価した良い実装]

### 🟡 改善提案 (任意)

- [リファクタリングやパフォーマンス改善の提案]

### 🔴 修正必須事項 (あれば)

- [バグや脆弱性のリスクがある箇所とその修正案]
```

## 4. ユーザーへの確認

提示後、「この提案に従って修正を行いますか？」と尋ね、ユーザーが承認した場合に実際の修正コードを適用します。
